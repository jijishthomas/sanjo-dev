# Sanjo Cine Mathew Website

Static personal brand website for `sanjo.in`. The site is generated by `scripts/build-site.mjs` and publishes static HTML, CSS, JS, images, redirects, sitemap, robots, CNAME, and blog article pages.

## Local Setup

```bash
npm install
npm run build
npm run preview
```

`npm run build` writes the generated site into the repository root, matching the current local workflow.

`npm run build:dist` writes a clean deployment artifact into `dist/`. This is what GitHub Actions uses.

## GitHub Pages Deployment

The workflow at `.github/workflows/pages.yml` deploys on pushes to `main`.

It uses:

- `actions/checkout`
- `actions/setup-node`
- `actions/configure-pages`
- `npm run build:dist`
- `actions/upload-pages-artifact`
- `actions/deploy-pages`

The deployment artifact is `dist/`. For the GitHub project page, the workflow builds with `/sanjo-dev` as the public base path so CSS, JS, images, PDFs, and internal links resolve under:

```txt
https://jijishthomas.github.io/sanjo-dev/
```

For the custom domain, configure GitHub Pages for the repository, point DNS for `sanjo.in` to GitHub Pages as required by GitHub, and build with `SANJO_BASE_URL=https://sanjo.in`, no `SANJO_PUBLIC_BASE`, and `SANJO_CNAME=sanjo.in`.

## Build Output

This is a static multi-page site, not a Vite or React SPA. SEO-friendly routes are generated as folder `index.html` files, for example:

- `/blog-insights/`
- `/blog-insights/the-resilience-response/`
- `/programs/`
- `/book-consultation/`

The old `/blog/` route and old imported blog post URLs are generated as redirects to the canonical `/blog-insights/` routes.

## Blog JSON

Blog posts are loaded from:

```txt
data/blogPosts.json
```

To add a post, add one JSON object to that array. Do not add comments inside the JSON file.

Recommended fields:

```json
{
  "id": "short-stable-id",
  "title": "Post title",
  "slug": "post-url-slug",
  "excerpt": "Short summary for cards and SEO.",
  "date": "2026-06-11",
  "author": "Sanjo Cine Mathew",
  "category": "Life Skills",
  "tags": ["confidence", "growth"],
  "readTime": "4 min read",
  "featured": false,
  "editorPick": false,
  "editorsPick": false,
  "trending": false,
  "popular": false,
  "image": "/blog/images/post/example.jpg",
  "imageAlt": "Accessible image description",
  "content": "<p>Article body HTML.</p>"
}
```

Graceful defaults are applied during build:

- Missing image: fallback thumbnail treatment
- Missing `readTime`: estimated from content
- Missing category: `Insights`
- Missing tags: empty array
- Missing feature flags: `false`

## Blog Images

Put blog images under:

```txt
blog/images/post/
```

Use root-relative image paths in JSON, for example:

```txt
/blog/images/post/my-new-post.jpg
```

## Verification

Before deployment, run:

```bash
npm run build:dist
npm run preview
```

Then verify:

- `http://localhost:4173/`
- `http://localhost:4173/blog-insights/`
- `http://localhost:4173/blog-insights/the-resilience-response/`
- `http://localhost:4173/blog/`
- `http://localhost:4173/sitemap.xml`
- `http://localhost:4173/robots.txt`

Production sitemap and robots URLs use `https://sanjo.in`.
